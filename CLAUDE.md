# ブログ執筆の前提 (Blog Authoring Guide)

このリポジトリでブログ記事を書く・編集する際の前提をまとめる。詳細なテーマ案は `docs/blog-ideas.md` を参照。

## プロダクト文脈

- アプリ名: **Poker Bankroll Manager**
- 対象プレイヤー: **ライブポーカー**プレイヤー (オンラインではない)
- ブログのゴール: アプリへの自然流入を増やすこと
- ターゲット読者: グローバル (英語圏メイン、9言語展開で波及)
- 執筆方針: 感情的ペイン × 具体シナリオ × アプリ導線の自然さ

記事は**ライブ前提で書く**。ハンド数・時給・バンクロール目安・レート表記 ($1/$2, $2/$5, $5/$10 …) はすべてライブキャッシュゲームの水準に合わせる。オンラインの話をする場合は明示的に区別する。

**通貨はすべて米ドル ($) に統一する。** 円・ユーロ・ポンド等は使わない。日本語記事であってもレート・バンクロール・時給・バイイン等の金額は $ 表記。読者向けに補足が必要な場合のみ括弧書きで換算に触れる程度にとどめる。

## ディレクトリ構造

```
content/blog/<slug>/<lang>.md     # 記事本体 (Markdown + frontmatter)
public/blog/<slug>/thumbnail.png  # サムネイル画像
```

- `slug` は記事ごとに一意のケバブケース (例: `bankroll-challenge-1000-to-25k`)
- 一覧取得・ルーティングは `src/lib/blog.ts` と `src/app/[lang]/blog/**` 参照
- 並び順は `date` の降順 (`src/lib/blog.ts`)

## サポート言語

`src/lib/i18n/config.ts` で定義された 9 言語:

`ja, en, fr, es, de, pt, zh-CN, zh-TW, ko`

- **デフォルト/フォールバック言語は `en`** — 指定言語のファイルが無ければ `en.md` にフォールバックする (`src/lib/blog.ts`)
- 新規記事はまず `ja.md` と `en.md` を揃える。他言語は段階的に追加でよい
- 多言語展開前提で、文化固有のジョーク・国内限定トピックは避ける

## フロントマター

全言語ファイルで同じ構造を使う:

```yaml
---
title: "記事タイトル (その言語で)"
date: "YYYY-MM-DD"
description: "一覧・SEO用の説明 (その言語で、120〜160字目安)"
thumbnail: "/blog/<slug>/thumbnail.png"
---
```

- `date` は全言語で同一 (初出日)
- `thumbnail` は全言語で同一パス (1枚を共有)
- `title` / `description` はローカライズする

## トーン・文体

- 日本語記事: である調ではなく、**です・ます調 + 箇条書き + テーブル**中心
- 英語記事: 会話調の導入 → データ/フレームワーク → アクションの流れ
- 強調は `**太字**` を多用、見出しは `##` / `###` のみ (H1 はタイトルが自動)
- 「感情フック → 数字/根拠 → アプリ活用」の順で書く
- ライブの時給やバンクロール目安を出すときは **遠征費・タイムチャージ等の経費を引いた実質値**に言及する

## アプリ導線

記事内で Poker Bankroll Manager の具体機能 (セッション記録、KPI、AIコーチ、チャレンジ進捗など) を **自然に**参照する。CTA は末尾だけでなく、文脈に噛み合う箇所で短く挟む。宣伝色を強くしない。

## サムネイル画像

記事ごとに **1 枚だけ** 生成し、全言語で共有する。

### 配置とフォーマット

- パス: `public/blog/<slug>/thumbnail.png`(frontmatter の `thumbnail` と一致させる)
- 形式: **PNG**
- アスペクト比: **3:2** (既存 2 枚と揃える。横長。正方形・縦長は不可)
- ファイルサイズ: 約 **1.5〜2.5 MB** 目安 (既存と同程度。過剰に圧縮しない)

### ビジュアル方針

既存 2 枚 (`bankroll-management-guide`, `bankroll-challenge-1000-to-25k`) のトーンに揃える。

- **シネマティック & フォトリアル** — イラスト調・フラットデザイン・アイコン集は使わない
- **ムーディな暗めの背景** — ストーム、夜、暗がりなど。明るすぎる昼間シーンは避ける
- **カラーパレット** — ディープグリーン(フェルト/自然)、ゴールド(光/チップ)、クリムゾン(危険/アクセント)を中心に
- **中心モチーフはポーカーチップ** — カード・テーブル・ラスベガスなども可だが、**必ずポーカーを想起させる要素を 1 つ以上入れる**
- **比喩的ストーリー構成** — 記事テーマを象徴するシーン (崖・頂き・嵐・成長グラフなど) を背景に置く。ベタな「チップとトランプの静物写真」だけにしない
- **光と霧の演出** — 斜光・リムライト・浮遊するエンバー/パーティクルで奥行きを出す

### プロンプトの書き方

画像生成モデル (Gemini / Midjourney / DALL-E 等) に渡すプロンプトは **英語で書く**。日本語プロンプトは精度が落ちやすい。

- シーン → モチーフ → 光 → カラー → スタイル → アスペクト比 の順で記述
- 記事のキーメタファー (破産確率なら「崖」、チャレンジなら「ラスベガスまでの道」、管理術なら「積み上がるチップ+上昇グラフ」) を必ず含める
- 締めに `editorial poker-magazine photorealism, 3:2 aspect ratio` を付けるとトーンが揃いやすい

### 避けるもの

- 文字・ロゴ・UI モックの直接レンダリング(言語非依存であるべきため)
- 特定プレイヤー・著名人・既存ブランドを想起させる顔や要素
- 暴力・ギャンブル依存を煽る描写(破産・敗北を比喩で表現するのは可)
- 過剰に明るく爽やかなストックフォト調

## 執筆時の実務チェック

- [ ] `content/blog/<slug>/` を作成し、最低でも `ja.md` と `en.md` を置く
- [ ] `public/blog/<slug>/thumbnail.png` を配置 (PNG、3:2、既存のシネマティック調に揃っている)
- [ ] 全ファイルでフロントマターの `date` / `thumbnail` が揃っている
- [ ] ライブポーカーの前提 (ハンド数/時給/BI数) で書かれている
- [ ] 見出し階層が `##` / `###` のみ
- [ ] 既存記事 (`bankroll-management-guide`, `bankroll-challenge-1000-to-25k`) のトーンに揃っている
